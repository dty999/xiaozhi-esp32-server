# 第六阶段：移动端方案

> **目标**：提供移动端可用方案，优先 PWA 响应式设计，保留 uni-app 作为微信小程序兜底。
> **验证标准**：手机浏览器访问可用，添加到主屏幕后离线可用。

---

## 6.1 PWA 实现

### 6.1.1 Web App Manifest

**文件**：`src/app/manifest.ts`

```typescript
import type { MetadataRoute } from 'next';

export default function manifest(): MetadataRoute.Manifest {
  return {
    name: '控制台',
    short_name: '智控台',
    description: '小智ESP32服务管理平台',
    start_url: '/home',
    display: 'standalone',
    background_color: '#ffffff',
    theme_color: '#2563eb',
    icons: [
      { src: '/icon-192.png', sizes: '192x192', type: 'image/png' },
      { src: '/icon-512.png', sizes: '512x512', type: 'image/png' },
    ],
  };
}
```

### 6.1.2 Service Worker（@serwist/next）

已在 `next.config.ts` 中配置（见 P1），自动生成。

### 6.1.3 离线缓存策略

- API 响应：NetworkFirst（先网络，失败后缓存）
- 静态资源：CacheFirst（缓存优先）
- 图片：StaleWhileRevalidate

## 6.2 响应式适配

### Tailwind 断点策略

| 断点 | 宽度 | 适配 |
|:---|:---|:---|
| `sm` | 640px | 手机横屏 |
| `md` | 768px | 平板 |
| `lg` | 1024px | 桌面 |
| `xl` | 1280px | 大桌面 |

## 6.3 uni-app 共存策略

### 修改 uni-app 项目 API 地址

原 uni-app 项目修改 `env/.env.production` 中的 API 地址指向新后端：

```env
VITE_API_BASE_URL=https://your-domain.com
```

### 保留功能

- ✅ 微信小程序端保留（WiFi 配网等功能 Next.js PWA 无法替代）
- ✅ Android/iOS 原生安装包保留
- ❌ H5 端可废弃（由 PWA 替代）
